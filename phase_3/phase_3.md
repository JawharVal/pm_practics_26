### **Оценка проекта: Мобильное приложение для напоминания о платежах**

**Концептуальное описание проекта**:
Приложение для напоминаний о сроках оплаты счетов, аренды, налогов и подписок. Приложение позволяет пользователю легко отслеживать свои регулярные платежи и получать уведомления за день до даты их оплаты.

---

### **Use Cases**

1. **Регистрация в приложении**

   * **Предусловия**:

     * Пользователь открывает приложение.
     * Пользователь выбирает опцию "Зарегистрироваться".
   * **Основной поток**:

     * Пользователь вводит email и пароль.
     * Система проверяет, что email не зарегистрирован, а пароль соответствует требованиям безопасности.
     * Система отправляет код подтверждения на email.
     * Пользователь вводит код подтверждения.
     * Пользователь вводит имя.
   * **Результат**: Создан аккаунт пользователя.

2. **Добавление метода оплаты**

   * **Предусловия**:

     * Пользователь зарегистрирован в приложении.
   * **Основной поток**:

     * Пользователь вводит данные о способе оплаты.
     * Пользователь нажимает "Сохранить".
   * **Результат**: Метод оплаты добавлен в систему.

3. **Установка напоминания о платеже**

   * **Предусловия**:

     * Пользователь находится на странице "Мои платежи".
     * Пользователь нажал на "Добавить напоминание".
   * **Основной поток**:

     * Пользователь вводит сумму и описание платежа.
     * Пользователь выбирает дату и регулярность платежа.
     * Система подтверждает создание напоминания.
   * **Результат**: Напоминание о платеже установлено.

4. **Просмотр напоминаний**

   * **Предусловия**:

     * Пользователь зарегистрирован в приложении.
   * **Основной поток**:

     * Пользователь заходит в раздел "Напоминания".
     * Система отображает список предстоящих платежей.
   * **Результат**: Пользователь видит все свои напоминания.

5. **Напоминание о регулярных платежах**

   * **Предусловия**:

     * Приближается срок регулярного платежа.
   * **Основной поток**:

     * Приложение отправляет пуш-уведомление за день до срока платежа.
     * Пользователь отмечает платеж как выполненный.
   * **Результат**: Пользователь получает напоминание и выполняет оплату.

6. **Просмотр истории платежей**

   * **Предусловия**:

     * Пользователь зарегистрирован в приложении.
   * **Основной поток**:

     * Пользователь заходит в раздел "История".
     * Система отображает список всех оплаченных платежей с суммами и датами.
   * **Результат**: Пользователь видит полную историю своих платежей.

7. **Анализ финансовых привычек**

   * **Предусловия**:

     * Пользователь зафиксировал хотя бы один платеж.
   * **Основной поток**:

     * Пользователь открывает раздел "Аналитика".
     * Система строит график расходов и доходов пользователя.
   * **Результат**: Пользователь может увидеть свои финансовые привычки.
