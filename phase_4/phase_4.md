### **Фаза 4 (Мета) - Прогнозирование**

#### **Разбиение на задачи**

1. **Декомпозиция проекта на задачи**:
   С учетом описанных **Use Cases** и функционала для проекта **Напоминатель о платежах**, вот разбивка задач для **Фазы 5**:

   * **Реализация регистрации пользователя**
   * **Настройка метода оплаты**
   * **Создание функционала напоминания о платеже**
   * **Дизайн и интеграция UI**
   * **Интеграция с базой данных**
   * **Функция истории платежей**
   * **Настройка уведомлений о платежах**
   * **Настройки пользователя и уведомлений**

   Эти задачи (всего 20-25) критичны для завершения приложения, некоторые задачи могут быть дополнительно разбиты по мере необходимости.

2. **Утверждение состава команды с делением на специальности**:
   Состав команды, исходя из обсуждаемых ранее **ролей**:

   * **3 разработчика backend**: Для работы с API, серверной логикой и интеграцией.
   * **2 разработчика frontend**: Для проектирования и реализации интерфейса.
   * **2 тестировщика**: Для ручного и автоматизированного тестирования.
   * **3 аналитика**: Бизнес-аналитик, системный аналитик, старший аналитик.
   * **1 дизайнер**: Для UI/UX.
   * **1 проектный менеджер**: Для контроля над проектом и соблюдения сроков.

3. **Определение ресурсных требований для каждой задачи**:
   Вот как может выглядеть распределение ресурсов по задачам:

   * **Реализация регистрации пользователя**: 2 Backend разработчика, 1 Frontend разработчик
   * **Настройка метода оплаты**: 1 Backend разработчик, 1 Frontend разработчик
   * **Создание напоминаний**: 2 Backend разработчика, 1 Frontend разработчик
   * **Интеграция с базой данных**: 2 Backend разработчика
   * **UI дизайн и интеграция**: 1 Дизайнер, 1 Frontend разработчик
   * **История платежей и уведомления**: 2 Backend разработчика, 1 Frontend разработчик

4. **Определение связей предшествования между задачами**:

   * **Регистрация пользователя** должна быть выполнена до **Настройки метода оплаты**.
   * **Функция напоминаний о платеже** не может быть реализована до **Настройки метода оплаты**.
   * **UI дизайн и интеграция** должны быть выполнены параллельно с backend задачами, чтобы обеспечить синхронную работу frontend и backend.

5. **Решение задачи эвристическими методами**:
   Мы будем использовать **эвристики** и простые расчеты для оценки продолжительности задач и завершения. Это включает:

   * **Оптимистичные**, **пессимистичные** и **наиболее вероятные оценки** для завершения задач.
   * Применение этих оценок к задачам и определение общей продолжительности проекта.

#### **Трёхточечные оценки задач**

Для каждой задачи, на основе предыдущих фаз,  **примеры оценок**:

| **№** | **Задача**                      | **BG (Best Guess)** | **P (Pessimistic)** | **O (Optimistic)** | **WE (Weighted Estimate)** |
| ----- | ------------------------------- | ------------------- | ------------------- | ------------------ | -------------------------- |
| 1     | Регистрация пользователя        | 3ч                  | 5ч                  | 2ч                 | 3.2ч                       |
| 2     | Настройка метода оплаты         | 2ч                  | 4ч                  | 1ч                 | 2.5ч                       |
| 3     | Создание напоминаний о платежах | 4ч                  | 6ч                  | 3ч                 | 4.3ч                       |
| 4     | Интеграция с базой данных       | 5ч                  | 7ч                  | 4ч                 | 5.3ч                       |
| 5     | Дизайн UI и интеграция          | 6ч                  | 8ч                  | 5ч                 | 6.3ч                       |
| 6     | Уведомления о платежах          | 4ч                  | 6ч                  | 3ч                 | 4.3ч                       |

**Формула для расчёта взвешенных оценок**:
[
\text{WE} = \frac{\text{O} + 4*\text{BG} + \text{P}}{6}
]

### **Прогнозирование времени выполнения методом Монте-Карло**

Мы можем использовать **метод Монте-Карло**, используя ваши данные для выполнения задач. Вот шаги, которые мы будем выполнять:

1. **Генерация случайных значений для продолжительности задач**:

   * Для каждой задачи генерируем случайные значения продолжительности на основе её **оптимистичных**, **пессимистичных** и **наиболее вероятных** оценок.

2. **Симуляция выполнения задач**:

   * **Симуляция** выполнения каждой задачи с использованием **метода 3-точечных оценок**.
   * Модель даст нам **диапазон** возможных сроков для выполнения **Фазы 5** (с учетом неопределенности и рисков).

3. **Результаты симуляции**:

   * При использовании **Монте-Карло** мы можем получить **достоверность 80%** для прогнозируемого времени завершения **Фазы 5** (например, 2 недели).
   * Это будет учитывать **исторический показатель завершения задач** из предыдущих фаз, как показано в файле статистики.

**Шаги симуляции методом Монте-Карло**:

* Используем **оценки продолжительности задач** (оптимистичные, пессимистичные, наиболее вероятные) из таблицы и применяем их к **модели симуляции**, чтобы предсказать общую продолжительность проекта.
* Финальный **результат** даст нам **распределение вероятности**, сколько времени потребуется для выполнения **Фазы 5**.
